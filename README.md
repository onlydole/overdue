# Overdue

> Don't let your knowledge expire.

Overdue is a FastAPI app for tracking and reviewing knowledge entries ("volumes"). It combines a web interface and REST API with game mechanics (Dewey decay, XP, badges, streaks, ranks) so your library stays active instead of going stale.

## What It Includes

- Web UI with Jinja2 + HTMX (`/`, `/my-library`, `/shelves`, `/leaderboard`, `/how-to-play`)
- REST API under `/api` for librarians, volumes, shelves, catalog, reading-room health, and bulletins
- JWT-based authentication ("Library Card")
- Async SQLAlchemy + SQLite by default
- Bot librarians for simulated activity and leaderboard movement
- CLI for serving, seeding data, auth, stats, volumes, shelves, and bot management

## Recent Changes (Codebase Audit)

- JWT implementation is now based on `PyJWT` (`src/auth/library_card.py`, `src/auth/web_session.py`, `pyproject.toml`).
- Reading Room activity feed now shows the reviewing librarian and a link to the reviewed volume title (`src/web/dashboard.py`, `templates/dashboard.html`, `templates/partials/activity_feed.html`).
- Avatars are served from pre-rendered static SVG assets in `static/icons/` and generated by `scripts/build_icons.py` (`src/web/templates.py`, `src/game/avatars.py`).

## Quick Start

### Docker

```bash
git clone https://github.com/onlydole/overdue.git
cd overdue
docker compose up --build
```

Open `http://localhost:8000`.

### Local Python

```bash
git clone https://github.com/onlydole/overdue.git
cd overdue

python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -e ".[dev]"

uvicorn src.main:app --reload
```

Open:

- Web app: `http://localhost:8000`
- API docs: `http://localhost:8000/docs`

### CLI

```bash
overdue --help
overdue serve --reload
```

## API Quick Reference

| Endpoint | Purpose |
|---|---|
| `POST /api/librarians/register` | Register librarian |
| `POST /api/librarians/login` | Obtain library card (JWT) |
| `POST /api/librarians/refresh` | Refresh library card |
| `GET /api/librarians/leaderboard` | Leaderboard data |
| `POST /api/volumes/` | Create volume |
| `POST /api/volumes/{volume_id}/review` | Review volume |
| `GET /api/catalog/autocomplete` | Catalog autocomplete |
| `GET /api/reading-room/health` | Reading-room health snapshot |
| `GET /api/reading-room/overdue` | Overdue report |
| `POST /api/bulletins/` | Create bulletin subscription |

Full reference: [docs/api/endpoints.md](docs/api/endpoints.md)

## Configuration

Settings are loaded from environment variables prefixed with `OVERDUE_`.

| Variable | Default | Description |
|---|---|---|
| `OVERDUE_DATABASE_URL` | `sqlite+aiosqlite:///./overdue.db` | Database URL |
| `OVERDUE_SECRET_KEY` | `change-me-in-production` | JWT signing key |
| `OVERDUE_ALLOWED_ORIGINS` | `["*"]` | CORS allowlist |
| `OVERDUE_PORT` | `8000` | App port |
| `OVERDUE_WEBHOOK_SECRET` | `""` | Bulletin signature secret |
| `OVERDUE_DEMO_PASSWORD` | generated | Optional fixed password for seeded demo users |

On startup, Overdue auto-creates tables, applies simple idempotent column migrations, seeds demo data if the DB is empty, and simulates bot activity.

```text
  cli/          # Typer CLI
templates/      # Jinja2 templates and partials
static/         # CSS, JS, pre-rendered SVG assets
tests/          # Test suite
docs/           # Guides and API docs
```
